## Root Cause Analysis

The tests are failing because background jobs are not completing within the expected timeframe, leading to timeouts. This can be caused by increased system load, inefficient job processing logic, or insufficient timeout thresholds configured for background job execution. The current maximum execution time of 1800 seconds (30 minutes) is being exceeded under certain load conditions, indicating either the jobs themselves need optimisation or the timeout values need to be raised.

## Evidence

Errors such as `DFAcceptanceTest.StopTestException: Background job did not complete within specified amount of time.` and `The Maximum Execution time of 1800 seconds has been exceeded.` appear consistently across all 11 affected tests. The failures span Person Data Management features including employee purge, property retention, cyclic reference deletion, backup table cleanup, and incomplete record identification workflows.

## Impact Assessment

11 tests blocked since various dates ranging from 2025-09-12 to 2025-10-09, impacting all Person Data Management features including:

- Employee record purge and retention
- Global property management
- Cyclic reference handling in pay run results
- Backup table lifecycle management
- Incomplete record identification

## Suggested Fix

1. **DevOps:** Increase the timeout duration for background jobs beyond the current 1800-second ceiling to accommodate peak-load scenarios.
2. **Dev:** Profile and optimise the background job processing logic to reduce execution time, particularly for large datasets and cyclic reference resolution.
3. **QA:** Simulate realistic load conditions in test environments to verify that jobs complete within the expected timeframe under representative workloads.
